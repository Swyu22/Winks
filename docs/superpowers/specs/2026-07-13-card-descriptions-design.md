# 卡片一句话简介与复制按钮改版设计

日期：2026-07-13

## 背景

当前链接卡片仅展示标题和标签，用户需要打开链接后才能进一步判断用途；复制链接按钮同时展示图标和文字，占据了较多卡片底部空间。

本轮改动在不改变现有信息架构和分类元数据合同的前提下：

- 将复制链接按钮调整为左下角 `28px × 28px` 的纯图标按钮。
- 在标题与标签之间增加最多 15 个字符的一句话简介。
- 简介允许为空；为空时仍保留一行高度，保证卡片网格的纵向节奏一致。
- 为当前生产库中的 36 条链接补齐客观、功能型简介。

## 交互与视觉约定

### 复制按钮

- 固定尺寸为 `28px × 28px`，位置保持卡片左下角并与内容左边缘对齐。
- 默认只显示复制图标，不显示“复制链接”文字。
- 复制成功后短暂切换为勾选图标和成功色。
- 保留动态 `aria-label`、`title`、键盘焦点样式和 `aria-live`，纯图标不牺牲可访问性。
- 图标使用项目既有的 `lucide-react`，不增加依赖。

### 一句话简介

- 展示位置：标题之后、标签之前。
- 字号：`text-sm`，比标签的 `text-xs` 大一级。
- 单行展示，超出容器时截断，不能撑高单张卡片。
- 空值不展示占位文案，但使用固定最小高度保留一行空白。
- 新增或编辑链接时为可选手工输入项，输入上限 15 个字符。

## 数据模型

为 `public.links` 增加独立可空字段：

```sql
description text null
```

数据库约束：

```sql
description is null
or (
  description = btrim(description)
  and char_length(description) between 1 and 15
)
```

选择独立列而不是继续写入 `category` 元数据，原因如下：

- `category` 已承担分类、标签和看板兼容编码，继续扩展会放大解析与迁移风险。
- 简介是明确的一等展示字段，独立列便于查询、校验和后续维护。
- 可空列兼容旧客户端和未填写简介的新卡片。

前端读取时将 `null` 或缺失字段归一化为 `''`；保存时先去除首尾空白，空字符串写回 `null`。输入控件与数据库共同保证最多 15 个字符。

## 缓存与兼容

- 本地缓存键由 `winks:links:v1` 升级为 `winks:links:v2`，避免旧形状缓存掩盖数据库简介。
- 旧数据、旧缓存或 Supabase 返回中缺少字段时，`hydrateLink` 统一补为 `''`。
- 新客户端把空简介保存为 `null`，旧客户端未发送该列时数据库继续接受。
- 不修改 `category` 编解码、默认看板、PIN 或 Vite `base` 等现有合同。

## 发布顺序

1. 完成迁移、前端与测试，并在本地全量验收。
2. 提交并推送源码，生产页面暂不更新。
3. 将迁移应用到已链接的 Supabase 项目并查询验证。
4. 注入生产 Supabase 环境变量执行 GitHub Pages 部署。
5. 对生产页面执行桌面端、移动端和数据回读验证。

若数据库迁移无法确认成功，停止前端生产部署，避免新客户端访问不存在的列。

## 验收标准

- 36 条现有记录全部有合法简介，长度为 1 至 15 个字符且无首尾空白。
- 新增链接可以不填简介，编辑时可以清空简介。
- 卡片简介位于标题和标签之间；空简介仍保留一行。
- 复制按钮在桌面和移动视口均为 `28px × 28px`，只显示图标且可键盘操作。
- 单元测试、ESLint、生产构建、部署环境检查全部通过。
- 生产页面可加载，Supabase 请求成功，无控制台错误和明显布局错位。
