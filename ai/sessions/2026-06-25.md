# 2026-06-25 — 加载性能 / favicon 命中率 / 点击计数排序 / 未分类置底

**模型**：Claude Opus 4.8（ultracode：多 Agent workflow 理解 + 对抗式审查）
**分支**：`main`
**版本**：`v1.2.0` → `v1.3.0`

## 1. 需求（用户原话拆解）

1. 线上页面与数据库加载慢，打开后要等一会才出内容。
2. 很多链接的 favicon/缩略图抓不到，只显示缺省图标（例：withoa）。
3. 没有排序——希望加点击计数器，点击多的链接排序上升。
4. 左侧 sidebar 的「未分类」排到最底部。
5. 改完审查/检验代码、修问题并优化，然后 commit / push / 上线。

## 2. 诊断（4 Agent 并行理解 + 人工精读，结论一致）

- **慢**：串行瀑布（HTML→入口JS→运行时动态 import Supabase SDK→createClient→查询→渲染）+ 零缓存 + 无 preconnect + 免费层 6 天 keepalive 余量不足导致冷启动。
- **favicon**：单一来源 Google s2，无兜底链；Google 未收录的站（withoa）直接 404 → 掉首字母。
- **排序/计数**：`filteredLinks` 只过滤不排序；schema 无 clicks 列；卡片 `<a>` 无打开事件。
- **未分类**：`collectClassificationsFromLinks` 把未分类前置（与诉求相反）。

## 3. 决策（向用户确认）

- **点击计数存储 = 方案 A 独立 `clicks` 列**（见 [ADR-0004](../../docs/30-decisions/adr-0004-clicks-as-dedicated-column.md)）。
- **历史改动 = 先单独提交**：工作区有上一会话（2026-06-14 审计）遗留的 14 个未提交改动，先作为独立 commit（`26cb2a6`），本次功能再单独 commit。

## 4. 实现

- **性能**：localStorage SWR 缓存（秒出 + 后台 revalidate，读路径经幂等 `hydrateLink`）；`supabaseClient` 模块加载即 `warmSupabase()`（preconnect + 预拉 SDK chunk）；`LinkGridSkeleton` 骨架替代 spinner；vite `manualChunks` 拆 React vendor（入口 chunk 56.5KB→12.3KB gz）；keepalive 6 天→3 天。
- **favicon**：`getFaviconCandidates` 多源级联（Google s2 sz=128 → 直连 `/favicon.ico` → DuckDuckGo），`LinkCard` onError 逐级降级。
- **点击计数**：`links.clicks` 列；`hydrateLink` 兜底；`handleOpenLink` 乐观自增 + `persistClick` 容错持久化（非 PIN、列缺失静默降级）；`filteredLinks` 按 clicks 降序、created_at 兜底。
- **未分类置底**：`sortClassificationsUncategorizedLast` 仅供侧栏渲染（`displayClassifications`）。

## 5. 对抗式审查（19 Agent，4 维 + 逐条验证）确认 5 条，处理：

- **HIGH（已修）**：未分类置底排序污染了删分类回退与新建默认分类逻辑 → 拆分 `activeClassifications`（未分类在前，供逻辑）与 `displayClassifications`（置底，供侧栏渲染）。
- **LOW（已修）**：排序比较器对不可解析时间戳返回 NaN → `Number.isFinite` 兜底。
- **LOW（已修）**：SWR 缓存读取绕过 hydrateLink → `readLinksCache` 内 `map(hydrateLink)`（幂等）。
- **NIT（按设计保留）**：快速双击丢增量（ADR-0004 已记录）；点击后卡片即时重排（热门板设计、新标签页打开不可感知）。
